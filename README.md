# Text-to-SQL Agent using LangGraph

A natural language to SQL application that converts user questions into executable SQL queries. The project uses LangGraph to orchestrate the workflow, FAISS for schema retrieval, and Llama 3.3 70B (via Groq) for SQL generation and answer synthesis.

## Overview

Instead of providing the entire database schema to the LLM, the application first retrieves only the relevant tables using vector similarity search. The selected schema is then used to generate SQL queries, validate them, execute them on a SQLite database, and return a natural language explanation of the results.

The workflow also includes retry and repair logic to handle invalid SQL queries automatically.

---

## Features

* Natural language to SQL conversion
* Schema-aware retrieval using FAISS
* SQL validation before execution
* Automatic SQL correction and retry mechanism
* Query safety checks for destructive operations
* Conversational memory for follow-up questions
* Result explanation generated by the LLM
* Streamlit-based user interface
* FastAPI backend with Swagger documentation

## System Architecture

```text
User Question
      │
      ▼
Schema Retrieval
(FAISS Vector Store)
      │
      ▼
SQL Generation
(Llama 3.3 70B)
      │
      ▼
Validation Layer
      │
 ┌────┴────┐
 │         │
Pass     Fail
 │         │
 ▼         ▼
Execute   Repair SQL
 │         │
 └────┬────┘
      ▼
Answer Generation
      │
      ▼
Final Response
```

## Installation

### Clone Repository

```bash
git clone https://github.com/<your-username>/sql-agent.git
cd sql-agent
```

### Create Virtual Environment

```bash
python -m venv venv

# Linux / Mac
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Configure Environment Variables

Create a `.env` file:

```env
GROQ_API_KEY=your_api_key
```

---

## Running the Application

### Streamlit UI

```bash
streamlit run app.py
```

Open:

```text
http://localhost:8501
```

### FastAPI Backend

```bash
uvicorn api:app --reload
```

Open Swagger documentation:

```text
http://localhost:8000/docs
```

## Deployment

The FastAPI backend was deployed on Render and exposed through Swagger documentation.

The project runs successfully in local development. Free-tier cloud deployments may encounter memory limitations due to the embedding model and FAISS retrieval pipeline.
